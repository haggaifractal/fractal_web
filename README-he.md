# Fractal IT - אתר תדמית (B2B)

אתר B2B מודרני, מהיר במיוחד ומאובטח, דו-לשוני (אנגלית/עברית), שנבנה עבור חברת פרקטל מחשבים ותקשורת 2002 בע"מ.

## 🚀 טכנולוגיות (Tech Stack)
* **תשתית:** [Astro](https://astro.build/) (v6)
* **עיצוב:** [Tailwind CSS](https://tailwindcss.com/) (v3.4)
* **סביבת ריצה:** Node.js (>=22.12.0)
* **אחסון ופריסה:** Cloudflare Pages

## 🛠️ מדריך הפעלה מהיר

להרצת הפרויקט בסביבת פיתוח, ודא/י שמותקן לך Node.js בגרסת 22 ומעלה.

1. **התקנת חבילות ותלויות:**
   ```bash
   npm install
   ```
2. **הפעלת שרת הפיתוח:**
   ```bash
   npm run dev
   ```
3. **צפייה בדפדפן:**
   גש/י אל [http://localhost:4321](http://localhost:4321)

## 🏗️ ארכיטקטורה וניתוב דו-לשוני (i18n)

פרויקט זה דוגל בארכיטקטורה מודולרית דו-לשונית תוך שמירה קפדנית על עקרון **DRY (Don't Repeat Yourself)**.

* **עברית (RTL):** הקבצים ממוקמים בשורש הפרויקט (`src/pages/index.astro`, `services.astro` וכו').
* **אנגלית (LTR):** הקבצים ממוקמים תחת תיקיית המקור הייעודית `/en/` (למשל `src/pages/en/index.astro`).
* **מעטפות ותבניות בסיס:** שתי השפות חולקות Layout פנימי אחד (`src/layouts/Layout.astro`) המנהל את המעבר בין ימין-לשמאל לשמאל-לימין (`dir="rtl"` או `dir="ltr"`) בצורה אוטומטית לפי מאפיין השפה. מבני הדפים מחוברים לפדרציית רכיבים תחת `/src/templates` למניעת קוד כפול.
* **ניהול מילונים (i18n):** לשמירה על סדר ברמת אנטרפרייז, הטקסטים המרכזיים מקוטלגים תחת קבצי JSON ייעודיים (`/src/locales/en.json` ו-`/src/locales/he.json`). טקסטים משפטיים ארוכים במיוחד (כמו תנאי שימוש, סודיות ונגישות) נשמרים כקבצי **Markdown** קלים (סיומת `.md`) תחת `src/pages/`, העטופים ב-`LegalLayout.astro`. זה מאפשר לצוותי התוכן והמשפטים לערוך אותם ללא חשש לשבירת קוד או שגיאות תחביר.

## 🎨 מערכת עיצוב וחוקי סטיילינג

הפרויקט **לא** עושה שימוש בצבעים הגנריים והסטנדרטיים של Tailwind (כמו `bg-blue-500`). התצורה בנויה לשימוש בסביבת ערכות צבעים שהוגדרה מראש בהשראת Material Design תחת `tailwind.config.mjs`.

בזמן פיתוח יש להקפיד להשתמש אך ורק בשמות (Tokens) הבאים:
* **רקעים:** `bg-surface`, `bg-surface-container`, `bg-surface-container-lowest`
* **טקסט:** `text-on-surface`, `text-on-surface-variant`
* **ההדגשות צבע:** `bg-primary`, `text-primary`, `bg-primary-fixed`
* **פונטים:** גוף ופונט מרכזי `font-sans` (Heebo/Inter), וכותרות `font-headline` (Space Grotesk)

## 🔒 אבטחה וטפסים

* **Web3Forms:** קומפוננטת הטופס (`ContactSection.astro`) מחוברת לפרוטוקול Web3Forms על מנת להעביר פניות ישירות למייל הארגוני (`Office@fractal.co.il`) בלי נתוני ביניים שנאגרים במסד נתונים של פלטפורמת הענן. **שים לב:** יש לספק Access Key חוקי כדי שזה יעבוד (ראה את קובץ ה-`.env.example` או הזרק אותו ישירות כמשתנה סביבה באתר החי שפועל ב-Cloudflare Pages).
* **הגנות צד שרת ופרוטוקול הענן:** הגדרנו תצורת HSTS, הגנה מגניבת-מידע (XSS), והקפדה על Content-Security-Policy - CSP בצורה נוקשה ברמת ייצור (הגדרות בקובץ ה-`public/_headers`).
* **Zero-Trust מוחלט:** בכדי להימנע מדליפת כתובות IP ובעיות הקשורות לרגולציית ה-GDPR, אנו אוסרים על שימוש בגישת APIs חיצונית לסינון תמונות (למשל Google Favicons).

## 🖼️ ניהול לוגואים של לקוחות (Privacy by Design)

בכדי לעמוד בכל תנאי החוק (הגנת הפרטיות) ולמנוע דליפות IP, לוגואים של לקוחות **לא** נטענים במצב ייצור מסרברים חיצוניים (שרתים של גוגל). הם מאוחסנים בלוקאל (פרויקט המקור).

**כיצד מעדכנים או מוסיפים לוגואים חדשים:**
1. פתח/י את `scripts/download-logos.cjs`.
2. הוסף/י דומיין חדש (או ערוך/י ישן) במערך `domains`.
3. הרץ/י `node scripts/download-logos.cjs` בטרמינל. הקוד ישאב וישמור עבורך פביקונים באיכות גבוהה בנתיב המקומי `public/images/clients/`.
4. דחוף/י את התמונות לריפוזיטורי (Cloudflare יפיץ עותק).

## ♿ נגישות (WCAG 2.1 AA)

הפרויקט כולל פתרון נגישות מקומי שקוף תוך שימוש בספרייה עצמאית של `accessibility`. בבסיס שלה היא נטענת באופן מקומי במלואה למניעת בעיות חדירה לפרטיות של פלטפורמות (לדוגמה עבריינות UserWay או Nagish). התוסף משנה את תוויות התפריט אוטומטית בהתבסס על אטריביוט ה-`lang` והמעבר חלק. 

## 🛡️ תאימות לחוק הגנת הפרטיות (תיקון 13) ו-SLA

האתר מאמץ את גישת ה-"Privacy by Design" (פרטיות כברירת מחדל):
1. **הסכמה ככלי עובדתי לקוקיז:** הפעלנו באנר הסכמות קשיח (`CookieBanner.astro`). סקריפטים של אנליטיקה או מעקב ייטענו **אך ורק** אם המשתמש מקיש "קבל הכל", שישגר אימות אל ה-`localStorage`.
2. **הזכות להימחק מזיכרון הנתונים (30 ימי סף שירות - SLA):**
   * **נוהל אדמיניסטרטור IT:** בהינתן שהפניות זורמות ישירות ממערכת הטפסים ל-`Office@fractal.co.il`, הצוות מחויב להגדיר כלל באאוטלוק (Outlook Rule) המוחק לחלוטין (Hard Delete) אימיילים שנושאם "New Inquiry from Fractal Website" בתום 30 עד 60 יום ממועד קבלתם.
   * זה מונע את הצורך לכתוב מערכות ניקוי במסד נתונים, ומוודא שזמן האגירה עומד לרשות דרישת המחוקק לבדו.

## 📝 מערכת ניהול תוכן (Sveltia CMS)

הפרויקט כולל מערכת ניהול תוכן Headless משולבת (Sveltia CMS) הממפה ישירות מול מאגר ה-GitHub שלנו. זה מאפשר לעורכי תוכן ומשפטנים לעדכן מהר וממשקית קבצי תרגום (`.json`) ודפי הרשאות וחוקים (`.md`) - מבלי לכתוב שורת קוד.

* **אבטחה וחיבור (Zero-Trust):** פלטפורמות CMS סטנדרטיות (כמו Decap) נעזרות במקורות CDN (לדוגמה `unpkg.com`) כדי לאשר סמכות, מה שמכניס סכנת שרשרת אספקה (Supply Chain Vulnerability). בפרויקט זה ניטרלנו סכנה זו בעזרת הורדת סקריפט ה-`sveltia-cms.js` ישירות לפרויקט. כך מדיניות ה-CSP (בתוך `_headers`) נותרה אטומה לחלוטין נגד צד-שלישי.
* **הזדהות פנימית:** בסביבת החיות (Cloudflare Pages), אימות הניהול של הפאנל מבוצע דרך מפתחות גישה מרחוק בעזרת GitHub Personal Access Tokens קלאסיים או עדינים המסופקים בהרשאות ניהוליות ותוכן - המייצרים אישיות ארכיטקטונית חסרת Database בענן.
* **תצורת DRY אחידה:** קובץ התצורה של ה-CMS בנתיב `public/admin/config.yml` משתמש רבות למעשה ב**עוגני המפות** (YAML Anchors, הסימנים `&` ו-`*`) כדי לדאוג שעל כל שדה תוכן אחד הוא מוזן באופן מיידי בהתאמה זהה לחלוטין לאנגלית ולעברית (`en.json` ו-`he.json`). זה מוודא אמינות, חוסר-שכפול ותחזוקה מהירה ויעילה.

## 📦 פריסה (Deployment)

התהליך הוסב כך שיאפשר העלאה שקופה ומיידית על גבי שרתי ה-**Cloudflare Pages**.
כל דחיפה של קוד לבראנץ' `main` יפעיל הרצת שרת בניות (`npm run build`) ויפזר את הדור הבא ישירות מעמודת ה-`dist/` לגולשים חיים בקו הענן המוכר.

*הערה: קובץ `package.json` מכיל מתאמי Override כדי להבטיח מעבר וגישור מעל התנגשויות התלויות של שילובי Tailwind יחד לצד Astro בצורה בטוחה לגמרי בלי לפגום בסיווג הבטיחות בבסיס החבילה.*

## ⚖️ הערות משפטיות ותאימות רגולטורית (חובה לקרוא לפני פיתוח)

> **⚠️ אזהרת הגנת פרטיות ומסדי נתונים (חובה):**
> 1. **תשתית טפסים רדומה (Web3Forms):** תשתית טופס יצירת הקשר (Web3Forms) בנויה במלואה בתוך הקוד אך היא מוגדרת כעת כ**רדומה** (לא פעילה). כרגע היא אינה מעבדת או שולחת נתונים לצדדים שלישיים. **אם תבחר/י להפעיל את Web3Forms בעתיד באמצעות מפתחות API, חובה עליך להודיע להנהלה באופן מיידי.** יהיה צורך לעדכן אקטיבית את מדיניות הפרטיות כדי לחשוף במפורש מבחינה משפטית את עיבוד הנתונים על ידי צד שלישי זה.
> 2. **אנליטיקה ושיווק מותאם אישית (תיקון 13 / GDPR):** האינטגרציה מול Google Analytics (GA4) / Google Ads מיושמת תחת אילוץ פנימי של האתר הדורש **הסכמה אקטיבית קשיחה (Strict Opt-In)** לשמירה על פרטיות. חל איסור מוחלט על טעינת סקריפטים של מעקב לפני שהמערכת מוודאת שקיים הערך `cookie-consent === 'granted'` ב-`localStorage`. חל עליך איסור מפורש לעקוף או להסיר מנגנון זה במהלך פיתוחים עתידיים, לרבות הטמעת פיקסלים ב-`<head>`.
